🦅 Varys

Varys is an AI-powered data science assistant that lives inside JupyterLab. Chat with it, ask it to write code, run EDA, generate plots, review your notebook, or complete your code inline — all without leaving the notebook interface.

No Node.js required. No cloud account lock-in. Works with Anthropic, OpenAI, Google, Ollama (local), AWS Bedrock, Azure OpenAI, and OpenRouter.

JupyterLab 4+ Python 3.9+ TypeScript / React 7 LLM Providers Local-first with Ollama

What makes it different

🗃️

Live DataFrame Context

Queries your running kernel for every DataFrame's schema automatically — no more describing columns manually.

🔗

Composite Pipelines

Chain multiple skills into a single command: load → EDA → annotate → README in one click.

🔀

Visual Diff View

See exactly which lines were added (green) and removed (red) before accepting any AI edit.

🧠

File-Driven Skills

Add new AI capabilities by dropping a SKILL.md file — no Python code changes required.

🛡️

Reproducibility Guardian

Catches missing random_state, hardcoded paths, data leakage risks on every cell run.

📦

pip install & done

Pre-built frontend bundle ships with the Python package. No Node.js, no build steps needed.

📦 Installation

Prerequisites

  • Python 3.9+
  • JupyterLab 4+
  • A virtual environment (recommended)
  • An API key for at least one LLM provider or Ollama running locally
✅ No Node.js required
The frontend bundle is pre-built and ships with the Python package. A plain pip install is all it takes.

Quick install

1

Create and activate a virtual environment

python3 -m venv .varys
source .varys/bin/activate   # Windows: .varys\Scripts\activate
2

Install Varys with your chosen provider

# Anthropic (Claude) — recommended
pip install "varys[anthropic] @ git+https://github.com/jmlb/varys"

# OpenAI (GPT / o-series)
pip install "varys[openai] @ git+https://github.com/jmlb/varys"

# Local models only (Ollama)
pip install "varys[ollama] @ git+https://github.com/jmlb/varys"

# Everything (all providers + RAG)
pip install "varys[all] @ git+https://github.com/jmlb/varys"
3

Configure your API key

Create or edit ~/.jupyter/varys.env:

~/.jupyter/varys.env 
# Provider selection
DS_CHAT_PROVIDER=ANTHROPIC
DS_COMPLETION_PROVIDER=ANTHROPIC

# API key
ANTHROPIC_API_KEY=sk-ant-...

# Model names
ANTHROPIC_CHAT_MODEL=claude-sonnet-4-6
ANTHROPIC_COMPLETION_MODEL=claude-haiku-4-5
4

Launch JupyterLab

jupyter lab

The 🦅 Varys icon appears in the left sidebar. Click it to open the assistant.

Verify installation

# Check frontend extension
jupyter labextension list
# Expected: varys v0.1.0  enabled  OK

# Check server extension
jupyter server extension list
# Expected: varys  enabled

Upgrade

pip install --upgrade "varys[all] @ git+https://github.com/jmlb/varys"

Uninstall

pip uninstall varys

🤖 LLM Providers

Varys supports 7 providers. Chat and completion can use different providers — route the expensive calls to a powerful cloud model and completions to a fast local one.

Supported providers

AnthropicANTHROPIC_API_KEYClaude models. Recommended for chat. Supports vision and tool-use streaming.
OpenAIOPENAI_API_KEYGPT-4o, o1, o3-mini, etc.

Per-task routing

Set different providers for chat and completion in ~/.jupyter/varys.env:

~/.jupyter/varys.env 
# Use Claude for chat (best quality), Ollama for completions (local, free)
DS_CHAT_PROVIDER=ANTHROPIC
DS_COMPLETION_PROVIDER=OLLAMA

Ollama setup

# Install Ollama
curl -fsSL https://ollama.com/install.sh | sh

# Pull a coder model
ollama pull qwen2.5-coder:1.5b-instruct   # fast, good for completion
ollama pull qwen2.5-coder:7b-instruct      # higher quality

# Start the server (keep running before launching JupyterLab)
ollama serve
~/.jupyter/varys.env 
DS_COMPLETION_PROVIDER=OLLAMA
OLLAMA_COMPLETION_MODEL=qwen2.5-coder:1.5b-instruct
OLLAMA_URL=                  # blank = http://localhost:11434
⚠️ Ollama must be running before JupyterLab
Start ollama serve in a terminal first, then launch JupyterLab. If Ollama is not running you'll see a connection error in the Varys chat.

Model Switcher

The model name shown below the chat input is a clickable button. Click it to open a popup listing all models in your configured model zoo. The selected model applies to chat requests only; the completion model is set separately in the Routing tab.

⚙️ Settings Panel

Click the ⚙ Settings button in the chat header to open the Settings panel. All changes are written to ~/.jupyter/varys.env and take effect immediately — no JupyterLab restart required.

Models tab

Contains a sub-tab for each provider. Each provider tab includes:

  • API key — masked input field
  • Chat model — model name used for chat requests
  • Completion model — model used for inline ghost-text suggestions
  • Model zoo — comma-separated list of model names to populate the model switcher dropdown

Routing tab

Sets which provider handles each task:

  • Chat provider dropdown (DS_CHAT_PROVIDER)
  • Completion provider dropdown + Token Limit input (DS_COMPLETION_PROVIDER, COMPLETION_MAX_TOKENS, default 128)

Skills tab

Lists every skill folder in ~/.jupyter-assistant/skills/ and the bundled skills:

  • Toggle — enable or disable a skill
  • Click a row — opens the skill editor with two tabs: SKILL.md and README.md
  • Ctrl+S — save the edited file
  • Refresh — reloads all skill files from disk without restarting

/ Commands tab

Lists every available slash command — both built-in and skill-defined — with their descriptions. Updated automatically when skills are added or removed.

📓 Notebook Operations

Varys can insert, modify, and delete cells in your notebook through the JupyterLab kernel API. All operations are shown in the Visual Diff View before you accept them.

Cell insertion modes

ModeBehaviourWhen to use

chatResponse appears in chat only, no cell changesAdvisory/explanatory skills

Safe auto-execution

The safe_operations skill defines which operations are allowed to run automatically after insertion:

  • ✅ Auto-execute: pd.read_csv, df.head(), df.describe(), print(), plotting
  • ❌ Require approval: to_csv, to_excel, inplace=True, model.fit(), API calls

🔀 Visual Diff View

After the AI applies changes in preview mode, a diff panel appears in the sidebar showing exactly what changed — green lines for additions, red lines for removals — before you accept.

Diff panel anatomy

  • Header — cell count, description, total +N / −N line stats
  • Per-cell accordion — click to expand. Shows badge (new / modified / deleted), cell type, position
  • Diff lines — green background for additions (+), red for removals (−), ±3 context lines around changes

Action buttons

ButtonAction
✓ AcceptRemove highlighting, keep changes, close diff panel
▶ Accept & RunAccept changes then immediately execute all code cells in the operation
✕ UndoRevert all cells to their state before the AI edit
ℹ️ Composite pipeline undo
When a composite pipeline runs, one diff panel covers all changes across all steps. Accept/Undo affects the entire pipeline atomically.

✨ Inline Completion

Ghost-text suggestions

As you type in a code cell, Varys automatically suggests completions as greyed-out ghost text. It uses the last 5 cells before the cursor as context, so it knows what DataFrames and functions are in scope.

  • Trigger: automatic, after a short debounce
  • Accept: Tab
  • Dismiss: Esc
  • Token limit: configurable in Settings → Routing (default 128)
  • Enable/disable: JupyterLab Settings → Inline Completer → Varys

Language detection

🧠 Skills System

Skills are markdown files that extend Varys's capabilities. They are injected into the LLM system prompt automatically based on context or an explicit slash command. No Python code changes needed.

Directory structure

~/.jupyter-assistant/skills/
└── my-skill/
    ├── SKILL.md       ← injected into the LLM system prompt
    └── README.md      ← human-readable docs (not sent to LLM)

Three-tier activation

TierWhen loadedHow to configure
Tier 1Every request — always injectedNo keywords in front matter
Tier 2When user message matches a keywordAdd keywords: [eda, explore] in front matter
Tier 3User types the skill's slash commandAdd command: /myskill in front matter

🗃️ DataFrame Auto-Detection

Before every chat request, Varys executes a small Python snippet silently in your kernel to collect the schema of every pandas DataFrame in memory. This is injected into the LLM prompt automatically — so the AI always knows what your data looks like without you having to describe it.

What the AI sees

## Live DataFrame Context (from kernel)

`df_sales` — 10,000 rows × 6 cols  (2.4 MB)
  product_id    int64
  product_name  object
  category      object
  price         float64
  quantity      int64
  sale_date     datetime64[ns]
  Sample (3 rows):
    {product_id=1, product_name='Widget A', price=29.99}

Privacy

Only schema metadata is sent to the LLM: column names, dtypes, and up to 3 sample rows (max 8 fields per row, truncated to 40 characters). No bulk data is transmitted. For full privacy, use Ollama — nothing leaves your machine.

Caching

The schema is cached by the maximum execution count across all cells. It is only re-fetched after you run a cell — rapid chat messages don't incur extra kernel round-trips.

📄 Report Generation

Generate a professional markdown report from your entire notebook — including all visualisations — with a single command.

Trigger

Type /report in the chat, or any of: "generate report", "create a summary report", "document my analysis".

Pipeline

1

Parse notebook

Reads all cells, outputs, and metadata from the .ipynb file.

2

Extract visualisations

All image/png cell outputs are extracted as base64 strings and replaced with ![Figure N] placeholders.

3

LLM generation

The notebook is sent to the LLM with the notebook-report-generation skill, which generates a structured markdown document.

4

Post-processing

Placeholder references are replaced with actual base64 image tags so images render inline in markdown previews.

5

Save & display

A timestamped .md file is saved alongside the notebook. A Report Card appears in chat with filename, word count, image count, and a download link.

💾 Memory System

Varys remembers your preferences per project, injecting them into every LLM request automatically.

Storage

Preferences are stored in .jupyter-assistant/memory/preferences.md — a human-readable markdown file. One file per project directory.

What gets remembered

  • Preferences saved with /learn ("always use seaborn", "no print statements")
  • Declined suggestions (so the AI won't propose the same thing again)
  • Style preferences learned from your workflow

Editing manually

Open .jupyter-assistant/memory/preferences.md in any editor and add preferences directly. The file is read fresh on every request.

preferences.md example 
# Varys — Memory

## User Preferences

- Always use seaborn for visualisations
- Prefer method chaining for pandas operations
- Never suggest in-place mutations

## Declined Suggestions

- 2025-01-15: Consolidate sklearn imports — user prefers imports at cell level

✏️ Creating Custom Skills

Adding a new skill is as simple as creating a folder and a markdown file — no Python changes required.

description: Does something useful
keywords: [trigger word, another trigger]

---

# My Skill

Everything below the front matter is injected into the LLM system prompt
whenever this skill is active.

## Rules
- Always do X
- Never do Y

  2. Click Refresh in the Skills tab of Settings

  4. Type your trigger keyword or slash command in the chat

⚙️ Creating Composite Skills

A composite skill chains multiple individual skills into a pipeline. Each step receives the notebook context updated by all previous steps.

SKILL.md format

~/.jupyter-assistant/skills/my-pipeline/SKILL.md 
---
composite: true
command: /mypipeline
description: Full analysis workflow
keywords: [run full workflow, end to end]
steps: [load-dataframe, eda, my-custom-skill]
step_prompts:
  - "Load the file {file_path} as df. Print shape and columns."
  - "Run EDA with visualisations for all numeric columns."
  - "Apply my custom skill to the results."
---

# My Analysis Pipeline

Orchestrates: Load → EDA → Custom analysis.
Extract the file path from the user message.

Variable substitution

VariableValue
{file_path}First file path found in the user's message
{user_message}The full original user message

Rules

  • All skills in steps must exist as folders in ~/.jupyter-assistant/skills/
  • step_prompts and steps must have the same length
  • Use cell_insertion_mode: preview on the composite skill itself
  • The final Accept/Undo panel covers all steps atomically

❓ FAQ

Does Varys send my notebook data to external servers?

Only the text content of your cells and their outputs (truncated) is sent to your configured LLM provider. Up to 3 sample rows per DataFrame are included for schema context. For full privacy, use Ollama — nothing leaves your machine.

Can I use Varys without an internet connection?

Yes — set both providers to OLLAMA and run ollama serve locally. No data leaves your machine.

Where is the config file stored?

At ~/.jupyter/varys.env. It's shared across all projects and all JupyterLab sessions. Changes are hot-reloaded on save.

My skill isn't being triggered. Why?

  1. Make sure the folder contains SKILL.md (exact capitalisation)
  2. Click Refresh in Settings → Skills tab
  3. Check that the skill is not listed in DS_SKILLS_DISABLED
  4. For Tier 2 skills, verify your message contains one of the keywords (case-insensitive)
  5. Check the / Commands tab — if your skill's command appears there, the skill is loaded

How do I undo an AI change after accepting it?

Once you click Accept, Varys no longer tracks the operation. Use JupyterLab's built-in undo (Ctrl+Z) inside a cell to revert content. For newly inserted cells, delete them manually.

What is the difference between /review and /ds-review?

/review is a classic code quality review — PEP8, naming, complexity, maintainability. /ds-review is a data science methodology review — it looks for data leakage, train/test contamination, inappropriate statistical tests, and missing reproducibility controls.

Can I use Varys with R or Julia notebooks?

The chat and inline completion features detect text/x-r-source and text/x-julia MIME types. Most skills are Python-oriented, but you can create custom skills for R or Julia workflows.

Can I run Python frontend changes without rebuilding?

Yes — Python server extension changes take effect immediately (no rebuild). Frontend TypeScript changes require jupyter labextension build . and a browser hard-refresh (Ctrl+Shift+R).

🔎 Troubleshooting

Varys icon not visible in sidebar

  • Hard-refresh: Ctrl+Shift+R (or Cmd+Shift+R on Mac)
  • Run jupyter labextension list — it should show varys enabled OK
  • If missing: pip install --force-reinstall "varys[all] @ git+https://github.com/jmlb/varys"

Error: 404 on /varys/task

  • The server extension is not loaded: jupyter server extension list
  • Reinstall: pip install --force-reinstall varys then restart JupyterLab

Anthropic model not found (404)

The model name in your .env is not valid. Use the current aliases:

~/.jupyter/varys.env 
ANTHROPIC_CHAT_MODEL=claude-sonnet-4-6
ANTHROPIC_COMPLETION_MODEL=claude-haiku-4-5

Ollama: "Request URL is missing protocol"

This happens when OLLAMA_URL is set to an empty string in .env. Either leave the key out entirely or set it to the full URL:

OLLAMA_URL=http://localhost:11434

Inline completion not appearing

  1. Open JupyterLab Settings → Inline Completer → ensure Varys is enabled
  2. Type at least 3 characters before a completion is requested
  3. Verify DS_COMPLETION_PROVIDER is set and the provider is reachable
  4. Check the browser console (F12) for errors

DataFrame schemas not in LLM context

  • Make sure the kernel is running and at least one cell has been executed
  • DataFrames must be in the top-level namespace (not inside a function)
  • Only pandas DataFrames are detected; numpy arrays and polars frames are not
